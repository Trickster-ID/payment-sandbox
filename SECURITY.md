# Security Policy

## Supported Versions

Security fixes are applied to the latest `master` branch state in this repository.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately to the maintainers before public disclosure.

Include at minimum:

- affected endpoint/module
- reproduction steps
- expected vs actual behavior
- impact assessment
- proof-of-concept (if available)

## Response Expectations

- Initial acknowledgment target: within 1 business day.
- Triage and severity assignment: within 3 business days.
- Fix target depends on severity (see `docs/iso/06-vulnerability-management.md`).

## Disclosure Guidelines

- Do not publicly disclose details until maintainers confirm remediation status.
- After fix is available, coordinated disclosure is allowed with technical accuracy.
